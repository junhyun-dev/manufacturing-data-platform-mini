Identity chain

Kafka coordinates and event IDs are preserved in the deterministic adapter. Its exact CSV SHA-256 becomes the existing pipeline's source_hash, so the batch and Iceberg retry contracts are reused instead of reimplemented.

Scope boundary

This is a bounded landing-to-batch bridge. It deliberately avoids continuous Spark Structured Streaming and direct Kafka-to-Iceberg writes because no window or latency pressure requires them yet.
